## 什么是图

### 定义

**图**（Graph）是一种非线性数据结构，由**顶点**（Vertex）的集合和**边**（Edge）的集合组成。

### 通俗理解

图就像是**社交网络**：
- **顶点** = 人（用户）
- **边** = 人与人之间的关系（好友关系）

```
╔══════════════════════════════════════════════════════╗
║          📊 现实生活中的图：社交网络                ║
╚══════════════════════════════════════════════════════╝

              👤 Alice
              │   │
              │   │
              │   │
         ┌────┘   └────┐
         │             │
    👤 Charlie      👤 Bob
         │             │
         │             │
         │        👤 David
         │
      👤 Eva

┌──────────────────────────────────────────────────┐
│ ✦ 顶点（节点）：Alice, Bob, Charlie, David, Eva │
│ ✦ 边（关系）：                                   │
│   • Alice — Charlie                             │
│   • Alice — Bob                                 │
│   • Bob — David                                 │
│   • Charlie — Eva                               │
└──────────────────────────────────────────────────┘
```

### 数学定义

```
╔══════════════════════════════════════════════════╗
║              📐 图的数学定义                     ║
╚══════════════════════════════════════════════════╝

    图 G = (V, E)

    其中：
    ┌────────────────────────────────────┐
    │ V = {v₁, v₂, ..., vₙ}              │
    │   ↓                                 │
    │   顶点集合（Vertex Set）            │
    │                                     │
    │ E = {(u, v) | u, v ∈ V}            │
    │   ↓                                 │
    │   边的集合（Edge Set）              │
    └────────────────────────────────────┘
```

## 图的基本组成

### 顶点（Vertex）

顶点是图中的基本单元，也称为**节点**（Node）。

```cpp
// 顶点示例
顶点 0, 顶点 1, 顶点 2, 顶点 3

在地图中：
- 顶点 = 城市
- 顶点 = 路口
- 顶点 = 地铁站
```

### 边（Edge）

边是连接两个顶点的线段，表示顶点之间的关系。

```cpp
// 边示例
(0, 1), (1, 2), (2, 3)

在地图中：
- 边 = 道路
- 边 = 航线
- 边 = 铁路线
```

### 可视化示例

```
╔══════════════════════════════════════════════════╗
║              🔷 简单图示例                      ║
╚══════════════════════════════════════════════════╝

    ●━━━━━━━●
    │ 0      │ 1
    │        │
    │        │
    │        │
    ●━━━━━━━●
    2        3

┌──────────────────────────────────────────────┐
│ 🔹 顶点集合 V = {0, 1, 2, 3}                 │
│ 🔹 边集合 E = {(0,1), (0,2), (1,3), (2,3)}  │
│ 🔹 顶点数 |V| = 4                            │
│ 🔹 边数 |E| = 4                               │
└──────────────────────────────────────────────┘
```

## 图的基本术语

### 1. 顶点和边

| 术语 | 英文 | 说明 |
|------|------|------|
| 顶点 | Vertex | 图中的节点 |
| 边 | Edge | 连接顶点的线 |
| 端点 | Endpoint | 边连接的两个顶点 |

### 2. 度（Degree）

**度**是指与一个顶点相连的边的数量。

```
╔══════════════════════════════════════════════════╗
║            📊 顶点的度（Degree）                  ║
╚══════════════════════════════════════════════════╝

    ●━━━━━━━●                    ┌──────────────┐
    │ 0      │ 1                   │ degree(0) = 2│ ← 连接 2 条边
    │        │                    └──────────────┘
    │        │
    │        │                    ┌──────────────┐
    ●━━━━━━━●                    │ degree(1) = 2│
    2        3                    └──────────────┘

    每个顶点的度数：
    ┌─────────────────────────────────┐
    │ degree(0) = 2  → 连接 1, 2      │
    │ degree(1) = 2  → 连接 0, 3      │
    │ degree(2) = 2  → 连接 0, 3      │
    │ degree(3) = 2  → 连接 1, 2      │
    └─────────────────────────────────┘
```

**有向图的度**：

| 类型 | 英文 | 说明 |
|------|------|------|
| 入度 | In-Degree | 指向该顶点的边数 |
| 出度 | Out-Degree | 从该顶点出发的边数 |

```
╔══════════════════════════════════════════════════╗
║         ➡️ 有向图：入度与出度                   ║
╚══════════════════════════════════════════════════╝

    ┌───┐     ┌───┐     ┌───┐
    │ 0 │────→│ 1 │────→│ 2 │
    └───┘     └─┬─┘     └─┬─┘
      ▲        │        │
      │        ▼        ▼
      │      ┌───┐   ┌───┐
      └──────│ 3 │───│   │
             └───┘   └───┘

    ┌────────────────────────────────────┐
    │ 顶点 1 的度分析：                   │
    │ ├── 入度 = 2 (从 0 和 3 进入)      │
    │ └── 出度 = 1 (指向 2)              │
    │                                    │
    │ 顶点 2 的度分析：                   │
    │ ├── 入度 = 1 (从 1 进入)           │
    │ └── 出度 = 0 (没有指向其他顶点)    │
    └────────────────────────────────────┘
```

### 3. 路径（Path）

路径是顶点的序列，其中相邻顶点之间有边相连。

```
╔══════════════════════════════════════════════════╗
║            🛤️ 路径（Path）示例                   ║
╚══════════════════════════════════════════════════╝

    ●━━━●━━━●
    │ 0 │ 1 │ 2              从 0 到 4 的可能路径：
    │   │   │
    │   │   │                ┌─────────────────────┐
    │   │   │                │ 路径 1: 0→1→2→4      │
    ●━━━●   │                │ 长度 = 3 (3条边)    │
    3       │ 4              ├─────────────────────┤
            │                │ 路径 2: 0→3→4        │
            └──┐             │ 长度 = 2 (2条边)    │
               │              ├─────────────────────┤
               └──────────┐   │ 路径 3: 0→1→2→...  │
                          │   │ 不存在 (无法到达)   │
                          └───┴─────────────────────┘

┌──────────────────────────────────────────────────┐
│ 📌 路径长度 = 路径中边的数量（不是顶点数）      │
│                                                  │
│ 例如：0 → 1 → 2 → 4                             │
│   ├── 经过的顶点：0, 1, 2, 4 (4个)               │
│   └── 路径长度：3 (3条边)                        │
└──────────────────────────────────────────────────┘
```

### 4. 环路（Cycle）

环路是起点和终点相同的路径。

```
╔══════════════════════════════════════════════════╗
║            🔄 环路（Cycle）                      ║
╚══════════════════════════════════════════════════╝

    【有环图】              【无环图（树）】
    ┌─────────┐            ┌─────────┐
    │  ●───●  │            │    ●    │
    │  │   │  │            │   /|\   │
    │  │   │  │            │  ● ● ●  │
    │  │   │  │            │  ●      │
    │  ●───●  │            │         │
    │         │            │         │
    │ 环路：  │            │ 没有环  │
    │ 0→1→3→2│            │ 路径    │
    │   ↖___│              │         │
    └─────────┘            └─────────┘

    环路示例：0 → 1 → 3 → 2 → 0
    ┌──────────────────────────────────────┐
    │ ✦ 起点 = 终点 = 0                   │
    │ ✦ 途中不重复访问顶点                 │
    │ ✦ 至少包含 3 个顶点                  │
    └──────────────────────────────────────┘
```

### 5. 连通性

| 术语 | 说明 |
|------|------|
| **连通图** | 任意两个顶点之间都有路径 |
| **非连通图** | 存在无法互相到达的顶点对 |
| **连通分量** | 极大连通子图 |

```
╔══════════════════════════════════════════════════╗
║            🔗 连通性对比                        ║
╚══════════════════════════════════════════════════╝

    【连通图】              【非连通图】
    ┌──────────┐           ┌──────────┐
    │  ●───●───●           │  ●───●   │
    │  │   │   │           │          │  ●───●
    │  │   │   │           │          │
    │  ●───●───●           │          │
    │         │           │          │
    │任意两点 │           │分为两组 │
    │可达     │           │互不可达 │
    └──────────┘           └──────────┘

┌──────────────────────────────────────────────┐
│ ✦ 连通图：任意两个顶点之间都有路径          │
│ ✦ 非连通图：存在无法互相到达的顶点对        │
│ ✦ 连通分量：极大连通子图                    │
└──────────────────────────────────────────────┘
```

### 6. 权重（Weight）

权重是边上的数值，表示边的"代价"或"成本"。

```
╔══════════════════════════════════════════════════╗
║         ⚖️ 带权图（Weighted Graph）              ║
╚══════════════════════════════════════════════════╝

        ①───[5]───②
        │          │
       [7]       [3]
        │          │
        ③─────────④
           [4]

┌──────────────────────────────────────────────┐
│ 边的权重：                                   │
│   • ①↔② = 5                                │
│   • ①↔③ = 7                                │
│   • ②↔④ = 3                                │
│   • ③↔④ = 4                                │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│ 📍 权重的实际意义：                          │
│   ├── 道路长度（公里）                       │
│   ├── 旅行时间（分钟）                       │
│   ├── 通行费用（元）                         │
│   ├── 网络延迟（毫秒）                       │
│   └── 带宽成本（单位成本）                   │
└──────────────────────────────────────────────┘
```

## 图的类型

### 1. 无向图（Undirected Graph）

边没有方向的图。

```
示例：
    A --- B
    |     |
    |     |
    C --- D

特点：
- 边 (A, B) 和 (B, A) 相同
- 邻接矩阵对称
- 用途：好友关系、道路网络
```

### 2. 有向图（Directed Graph / Digraph）

边有方向的图。

```
示例：
    A --> B --> C
    ↑     ↓     ↓
    └----- D ---┘

特点：
- 边 (A, B) 和 (B, A) 不同
- 邻接矩阵不一定对称
- 用途：关注关系、网页链接、工作流
```

### 3. 带权图（Weighted Graph）

边带有权重的图。

```
示例：
    A --5-- B --3-- C
    |       |
    7       4
    |       |
    D-------+

特点：
- 每条边有权重
- 用途：最短路径、最小生成树
```

### 4. 无权图（Unweighted Graph）

边不带权重的图（所有边权重相同）。

```
示例：
    A --- B --- C
    |     |
    D-----+

特点：
- 所有边权重相同（通常为 1）
- 用途：连通性判断、可达性分析
```

### 5. 简单图（Simple Graph）

没有自环和重边的图。

```
简单图：
    0 --- 1 --- 2
    |         |
    3 ------- 4

特点：
- 没有自环（顶点到自己的边）
- 没有重边（两个顶点之间最多一条边）
```

### 6. 多重图（Multigraph）

允许有重边的图。

```
多重图：
    0 === 1 === 2
    ||     ||
    3 -----+

特点：
- 允许两个顶点之间有多条边
- 用途：航班路线（多个航班）
```

### 7. 完全图（Complete Graph）

任意两个顶点之间都有边的图。

```
╔══════════════════════════════════════════════════╗
║       💎 完全图 K4 (Complete Graph)              ║
╚══════════════════════════════════════════════════╝

            ●────────●
         ╱│  0     │  1│╲
        ╱ │         │   │ ╲
       ╱  │         │   │  ╲
      ●───●─────────●───●
     │ 3  │         │ 2 │
     │    │         │   │
     │    ╲─────────╱   │
     │      ╲     ╱     │
     ●───────●───────●
           所有顶点两两相连

┌──────────────────────────────────────────────┐
│ 📊 完全图性质：                               │
│   ├── n 个顶点的完全图有 n(n-1)/2 条边       │
│   ├── K4 (4个顶点) 有 6 条边                 │
│   ├── 每个顶点的度数 = n-1                   │
│   └── 用途：理论分析、算法测试                │
└──────────────────────────────────────────────┘
```

### 8. 二分图（Bipartite Graph）

顶点可以分为两个独立集合，边只在两个集合之间。

```
╔══════════════════════════════════════════════════╗
║       🎯 二分图（Bipartite Graph）                ║
╚══════════════════════════════════════════════════╝

    【A 组顶点】        【B 组顶点】
    ┌─────────┐        ┌─────────┐
    │  ○───○  │        │         │
    │  │   │  │────────│  ○───○  │
    │  │   │  │        │  │   │  │
    │  │   │  │--------│--│   │  │
    │  │   │  │        │  │   │  │
    │  ○───○  │        │  ○───○  │
    │  0 1 2  │        │  3 4 5  │
    └─────────┘        └─────────┘
       ←───────→
      跨组连接

┌──────────────────────────────────────────────┐
│ 🎲 二分图特点：                              │
│   ├── 顶点分为两个独立集合 A 和 B            │
│   ├── 边只连接不同组的顶点                   │
│   ├── 同组内没有边相连                       │
│   └── 用途：匹配问题、调度问题、任务分配    │
└──────────────────────────────────────────────┘

应用示例：
┌──────────────────────────────────────────────┐
│ 👫 男孩-女孩配对                             │
│ 👷 工人-任务分配                             │
│ 📚 学生-课程选课                             │
│ 🏠 公司-职位招聘                             │
└──────────────────────────────────────────────┘
```

### 9. 树（Tree）

连通且无环的无向图。

```
╔══════════════════════════════════════════════════╗
║          🌳 树（Tree）                          ║
╚══════════════════════════════════════════════════╝

              ○ 0 (根节点)
             ╱│╲
            ╱ │ ╲
           ╱  │  ╲
          ○   ○   ○
         1    2   3
        ╱╲    │
       ╱  ╲   │
      ○    ○  ○
     4    5  6

┌──────────────────────────────────────────────┐
│ 🌲 树的特点：                                 │
│   ├── 连通且无环                             │
│   ├── n 个顶点，n-1 条边                     │
│   ├── 任意两顶点间只有唯一路径               │
│   ├── 有一个根节点                           │
│   └── 用途：文件系统、组织结构、DOM树        │
└──────────────────────────────────────────────┘

树 vs 普通图：
┌──────────────┬──────────────┐
│     树       │    普通图    │
├──────────────┼──────────────┤
│ 无环         │ 可能有环     │
│ n-1 条边     │ 任意条边     │
│ 层次结构     │ 网状结构     │
│ 唯一路径     │ 多条路径     │
└──────────────┴──────────────┘
```

### 10. 有向无环图（DAG）

没有环路的有向图。

```
╔══════════════════════════════════════════════════╗
║       📈 有向无环图（DAG）                       ║
╚══════════════════════════════════════════════════╝

    ┌───┐     ┌───┐     ┌───┐
    │ 0 │────→│ 1 │────→│ 2 │
    └───┘     └─┬─┘     └─┬─┘
                │        │
                ▼        ▼
              ┌───┐   ┌───┐
              │ 3 │──→│ 4 │
              └───┘   └───┘

┌──────────────────────────────────────────────┐
│ 🎯 DAG 特点：                                │
│   ├── 有向图（边有方向）                     │
│   ├── 无环（不存在循环依赖）                 │
│   ├── 至少有一个入度为 0 的顶点（源点）      │
│   ├── 至少有一个出度为 0 的顶点（汇点）      │
│   └── 用途：任务调度、依赖关系、版本控制    │
└──────────────────────────────────────────────┘

实际应用：
┌──────────────────────────────────────────────┐
│ 📦 任务依赖：A→B→C (必须按顺序执行)         │
│ 🔧 编译依赖：模块导入关系                    │
│ 📚 课程先修：课程A是课程B的前置             │
│ 🏗️ 项目管理：工作流设计                     │
└──────────────────────────────────────────────┘
```

## 图的表示方法

### 1. 邻接矩阵（Adjacency Matrix）

使用 n×n 的二维数组表示图。

```
╔══════════════════════════════════════════════════╗
║     📊 邻接矩阵（Adjacency Matrix）              ║
╚══════════════════════════════════════════════════╝

    原图：                    邻接矩阵：
    ●━━━●                   ┌───┬───┬───┬───┐
    │ 0 │ 1                 │   │ 0 │ 1 │ 2 │ 3 │
    │   │                   ├───┼───┼───┼───┤
    ●━━━●                   │ 0 │ 0 │ 1 │ 1 │ 0 │ ← 0 与 1,2 相连
    2   3                   │ 1 │ 1 │ 0 │ 0 │ 1 │
                            │ 2 │ 1 │ 0 │ 0 │ 1 │
                            │ 3 │ 0 │ 1 │ 1 │ 0 │
                            └───┴───┴───┴───┘

┌──────────────────────────────────────────────┐
│ ✅ 优点：                                     │
│   ├── O(1) 时间检查边是否存在                │
│   ├── 实现简单直观                           │
│   ├── 适合稠密图                             │
│   └── 矩阵运算方便                           │
├──────────────────────────────────────────────┤
│ ❌ 缺点：                                     │
│   ├── O(V²) 空间复杂度                       │
│   ├── 不适合稀疏图                           │
│   └── 获取邻接点需要 O(V) 时间               │
└──────────────────────────────────────────────┘
```

### 2. 邻接表（Adjacency List）

使用数组+链表表示图。

```
╔══════════════════════════════════════════════════╗
║     📋 邻接表（Adjacency List）                 ║
╚══════════════════════════════════════════════════╝

    原图：                    邻接表：
    ●━━━●                   0 → [1, 2]
    │ 0 │ 1                 1 → [0, 3]
    │   │                   2 → [0, 3]
    ●━━━●                   3 → [1, 2]
    2   3

    ┌────────────────────────────────────┐
    │ 数组索引 → 邻接点链表               │
    ├────────────────────────────────────┤
    │ adj[0] → {1, 2}                   │
    │ adj[1] → {0, 3}                   │
    │ adj[2] → {0, 3}                   │
    │ adj[3] → {1, 2}                   │
    └────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│ ✅ 优点：                                     │
│   ├── O(V+E) 空间复杂度                      │
│   ├── 适合稀疏图                             │
│   ├── 快速遍历邻接点                         │
│   └── 易于添加/删除顶点                     │
├──────────────────────────────────────────────┤
│ ❌ 缺点：                                     │
│   ├── O(degree) 时间检查边是否存在           │
│   ├── 实现稍复杂                             │
│   └── 缓存不友好                             │
└──────────────────────────────────────────────┘

空间对比示例（V=1000, E=5000）：
```

## 常见算法

### 1. 遍历算法

| 算法 | 全称 | 特点 |
|------|------|------|
| **BFS** | 广度优先搜索 | 层级遍历，找最短路径 |
| **DFS** | 深度优先搜索 | 深度探索，找路径 |

```cpp
// BFS 示例
void BFS(int start) {
    std::queue<int> q;
    q.push(start);
    visited[start] = true;

    while (!q.empty()) {
        int u = q.front();
        q.pop();
        // 处理 u
        for (int v : adj[u]) {
            if (!visited[v]) {
                visited[v] = true;
                q.push(v);
            }
        }
    }
}
```

### 2. 最短路径算法

| 算法 | 适用图 | 时间复杂度 |
|------|--------|-----------|
| **BFS** | 无权图 | O(V + E) |
| **Dijkstra** | 非负权图 | O((V + E) log V) |
| **Bellman-Ford** | 允许负权 | O(VE) |
| **Floyd-Warshall** | 所有点对 | O(V³) |

```cpp
// Dijkstra 算法示例
void dijkstra(int start) {
    std::priority_queue<pair<int,int>> pq;
    pq.push({0, start});
    dist[start] = 0;

    while (!pq.empty()) {
        int u = pq.top().second;
        pq.pop();

        for (auto& edge : adj[u]) {
            int v = edge.first;
            int w = edge.second;
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}
```

### 3. 最小生成树算法

| 算法 | 时间复杂度 | 特点 |
|------|-----------|------|
| **Prim** | O((V + E) log V) | 适合稠密图 |
| **Kruskal** | O(E log E) | 适合稀疏图 |

## 图的基本操作

### 1. 添加顶点

```cpp
void addVertex() {
    adj.emplace_back();  // 添加新的空邻接表
    V++;
}
```

### 2. 添加边

```cpp
void addEdge(int u, int v) {
    adj[u].push_back(v);
    adj[v].push_back(u);  // 无向图
}
```

### 3. 删除边

```cpp
void removeEdge(int u, int v) {
    adj[u].erase(std::remove(adj[u].begin(), adj[u].end(), v), adj[u].end());
    adj[v].erase(std::remove(adj[v].begin(), adj[v].end(), u), adj[v].end());
}
```

### 4. 检查边是否存在

```cpp
bool hasEdge(int u, int v) {
    for (int neighbor : adj[u]) {
        if (neighbor == v) return true;
    }
    return false;
}
```

## 特殊的图

### 1. 稀疏图 vs 稠密图

| 类型 | 定义 | 边数 |
|------|------|------|
| 稀疏图 | E << V² | E < V²/10 |
| 稠密图 | E ≈ V² | E > V²/10 |

```
稀疏图：          稠密图：
0 --- 1          0 === 1
|                 |\ /|
2                 | X |
                  |/ \|
                  3 --- 4
```

### 2. 连通图

| 类型 | 定义 |
|------|------|
| 强连通图 | 有向图中，任意两个顶点可互相到达 |
| 弱连通图 | 有向图忽略方向后连通 |
| 连通图 | 无向图，任意顶点可达 |

### 3. 正则图

所有顶点的度都相同的图。

```
3-正则图（每个顶点度数为3）：
      0 --- 1
     /|\   /|\
    / | \ / | \
   4--+---+--2
    \ | / \ | /
     \|/   \|/
      3 --- 5
```
